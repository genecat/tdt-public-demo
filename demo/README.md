# TDT Demo

From the project root: `python3 tdt_cli.py demo/run.json` (see `run_demo.md`).

This demo shows how TDT evaluates the structural quality of outputs.

## What this demonstrates

- Strong vs weak vs failed responses
- How TDT surfaces issues like:
  - vagueness
  - instruction failure
  - low information density

## Why it matters

TDT does not check if something is true.

It checks if the output is structurally sound.
